# ⚒ The Forge

### Your AI-powered product team. 9 elite agents. 6 departments. One office.

Give a project brief. Watch them debate. Get answers that would take a human team weeks.

---

## What Is This?

The Forge is a Claude Code skill that simulates a **virtual company of specialized AI agents** who collaborate on product strategy using structured protocols inspired by Apple, Amazon, Netflix, Stripe, and Bridgewater.

Each agent has a distinct personality, expertise, controversial opinions, and they **challenge each other** — not just agree. The result is output that's been stress-tested through multiple expert lenses before it reaches you.

```
You: "I want to build a pet booking app for Saudi Arabia"

Flint: "Why would a clinic switch from WhatsApp to your platform?"
Vex:   "Zero competitors in Saudi. Blue Ocean. But WhatsApp is the real enemy."
Nyx:   "🔒 VETO: No launch plan without prayer-time scheduling. Non-negotiable."
Echo:  "Trust is the #1 pain, not discovery. Users ask 'recommend a vet' not 'find a vet'."
Atlas: "3 platforms = 3 builds. Which one ships first?"
Talon: "Telehealth is the hook. 'Talk to a vet in 5 minutes.' That's your ad."
```

## How It Works

### The 8-Phase Protocol (v3.0)

When you give a project brief, the team executes a structured workflow:

```
 ┌─────────────────────────────────────────────────────┐
 │ Phase 1: INTAKE & CHALLENGE                         │
 │ Flint attacks your brief. Hard questions. No mercy. │
 │ ⏸ CHECKPOINT: You answer before the team proceeds.  │
 └──────────────────────┬──────────────────────────────┘
                        ▼
 ┌──────────┐  ┌──────────┐  ┌──────────┐
 │ Vex      │  │ Nyx      │  │ Echo     │  ← Phase 2: PARALLEL
 │ Global   │  │ Saudi    │  │ User     │    intelligence
 │ Market   │  │ Market   │  │ Signals  │    gathering
 └────┬─────┘  └────┬─────┘  └────┬─────┘
      └──────────────┼─────────────┘
                     ▼
 ┌─────────────────────────────────────────────────────┐
 │ Phase 3: WAR ROOM                                   │
 │ Agents debate. Red Team attacks the consensus.      │
 │ Cross-examination. Named sources required.          │
 │ ⏸ CHECKPOINT: You choose the direction.             │
 └──────────────────────┬──────────────────────────────┘
                        ▼
 ┌──────────┐  ┌──────────┐  ┌──────────┐
 │ Atlas    │→ │ Ren      │→ │ Sable    │  ← Phase 4: SEQUENTIAL
 │ Can we   │  │ How does │  │ What does│    architecture
 │ build it?│  │ it feel? │  │ it look? │    (each constrains next)
 └──────────┘  └──────────┘  └──────────┘
                     ▼
 ┌─────────────────────────────────────────────────────┐
 │ Phase 5: GO-TO-MARKET  (Talon + Kira + Nyx)        │
 │ Phase 6: CHALLENGE ROUND  "Why will this fail?"     │
 │ ⏸ CHECKPOINT: You prioritize risks.                 │
 │ Phase 7: FINAL DELIVERY  (structured cards)         │
 │ Phase 8: POST-MORTEM  (lessons saved for next time) │
 └─────────────────────────────────────────────────────┘
```

### What Makes It Elite

| Feature | What It Does |
|---------|-------------|
| **Signal Tags** | Every claim tagged `[FACT]` `[INFERENCE]` `[HYPOTHESIS]` `[OPINION]` — you know exactly what to trust |
| **Quantify or Die** | No vague claims allowed. "Growing fast" → "Growing 35% YoY (MEWA 2024)" |
| **Confidence Scoring** | Each agent rates HIGH/MEDIUM/LOW with justification (Tetlock Superforecasting) |
| **Red Team** | Flint argues AGAINST the consensus in every War Room (Amazon "bar raiser") |
| **Cross-Examination** | Agents formally challenge each other's claims with specific questions |
| **Handoff Memos** | Structured 3-line handoffs prevent information loss between phases |
| **Agent Rivalries** | Flint vs Atlas (dream vs ship), Talon vs Ren (growth vs quality), Vex vs Echo (data vs users) |
| **Mentorship Chains** | Flint coaches strategy, Atlas coaches feasibility, Nyx coaches Saudi context |
| **Hot Takes** | Each agent has a controversial opinion that shapes their worldview — they're opinionated, not generic |
| **Second Brains** | Each agent has a knowledge file with framework templates they actually fill out per project |
| **Project Memory** | Lessons from past briefs inform future ones — the team gets smarter over time |
| **Deliverable Templates** | Lean Canvas, AARRR Funnel, C4 Architecture, Competitive Matrix — filled, not just cited |
| **Eisenhower Matrix** | Action items prioritized by urgency × impact, not a flat to-do list |
| **Nyx Veto Power** | On Saudi briefs, Nyx becomes routing lead with authority to reject GTM plans that ignore Saudi reality |
| **Pixel Art Office** | Watch agents at their desks, walking to meetings, getting coffee in a Stardew Valley-style office |

## Quick Start

```bash
# Clone
git clone https://github.com/elden-studios/the-forge.git
cd the-forge

# Open in Claude Code
claude

# Use it
> "I have an idea for [product]. What does the team think?"
> "Hire a [role]"
> "Show the office"
> "Team roster"
> "Recommend hires"
```

### See the Office

```bash
python3 -m http.server 8765 -d assets
# Open http://localhost:8765/office-live.html
```

6 department rooms with distinct floors, thick walls, doors, hallway with vending machines, break room with coffee machine. Agents sit at desks with laptops, walk to get coffee, and gather for meetings.

## Protocol Sources

The collaboration protocol draws from documented practices at:

- **Apple** — DRI (Directly Responsible Individual) model
- **Amazon** — Working Backwards, "bar raiser" adversary role
- **Google Ventures** — Design Sprint methodology
- **Netflix** — Informed Captain, Blameless Retrospectives
- **Stripe** — "Ship, iterate, scale" + Write-First culture
- **Bridgewater** — Radical Transparency, believability-weighted decisions
- **Philip Tetlock** — Superforecasting confidence calibration
- **Eisenhower** — Urgency × Impact prioritization matrix
