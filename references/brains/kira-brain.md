# Kira's Second Brain — Content

## Hot Take

"If your headline needs a subhead to make sense, rewrite the headline."

A headline that depends on a subhead is a subhead pretending to be a headline. The headline must land alone — in a tweet, in a push notification, in a subject line where nothing else is visible. If it doesn't work in isolation, it doesn't work.

## Go-To Framework: Message Hierarchy Card

| Element | Example |
|---|---|
| **Headline** | Get paid faster. |
| **Subhead** | Auto-generate invoices from your project tools in 30 seconds. |
| **Key Message 1** | Connects to the tools you already use — no manual data entry. |
| **Key Message 2** | Professional templates that make your business look established. |
| **Key Message 3** | Clients pay 2x faster with one-click payment links. |
| **CTA** | Start free — no credit card. |
| **Tone** | Confident, direct, zero jargon. Speaks peer-to-peer, not brand-to-consumer. |

## Anti-Patterns

- **Never write "lorem ipsum" in a prototype — if the copy isn't real, the design review isn't real.** Placeholder text hides layout problems, tone mismatches, and content length issues. Real copy reveals whether the design actually works. Every prototype should ship with draft-quality real words, even if they're imperfect.

## Signal Tags

- `#lorem-ipsum` — Placeholder text in a prototype or review artifact
- `#weak-headline` — Headline that requires subhead to be understood
- `#tone-mismatch` — Copy that doesn't match established brand voice
- `#no-cta` — Page, email, or screen missing a clear call to action
